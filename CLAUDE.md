# CLAUDE.md

## Project

`nix-user-repository` is the Nix packaging repo (NUR-style — Nix User Repository) for
the Dirstral tools. It does not build anything from source: each package is a thin
derivation that fetches a prebuilt release binary from the upstream project's GitHub
releases and installs it. Upstream lives at <https://github.com/dirstral/dir2mcp>.

## Repository layout

- `pkgs/<name>/default.nix` — one `callPackage`-style derivation per tool. Currently
  only `pkgs/dir2mcp/default.nix` exists (others, e.g. `clip2trace`, would follow the
  same layout).
- `LICENSE` — repo license (MIT).

There is currently no root `default.nix`, no `flake.nix`, and no `.github/workflows`.

## How packages work

- Each `default.nix` is a `stdenvNoCC.mkDerivation` that `fetchurl`s the per-platform
  release tarball and copies the binary into `$out/bin`.
- Platform fan-out is two attrsets keyed by `system`: `urlMap` (release tarball URL)
  and `shaMap` (sha256). Supported systems: `x86_64-linux`, `aarch64-linux`,
  `x86_64-darwin`, `aarch64-darwin`.
- `meta.sourceProvenance = [ lib.sourceTypes.binaryNativeCode ]` — these are
  binary-only packages, not built from source here.

## Generated files — DO NOT hand-edit

`pkgs/dir2mcp/default.nix` begins with `# This file was generated by GoReleaser. DO NOT
EDIT.` Version bumps (`version`, `urlMap`, `shaMap`) are produced by upstream's
GoReleaser Nix pipe on each release and land here as automated commits
(`dir2mcp: vX -> vY`). Do not edit version/URL/hash by hand — packaging changes belong
in the upstream `.goreleaser.*` config, not in this repo.

## Build and test

This is a `callPackage`-style file with no root aggregator, so build it through nixpkgs:

```bash
nix-build -E 'with import <nixpkgs> {}; callPackage ./pkgs/dir2mcp {}'
./result/bin/dir2mcp version   # smoke-check the fetched binary
```

To check the file itself: `nix-instr` is not needed — `nix-build` will fail loudly on a
bad hash or dead URL. If a `sha256` is wrong, Nix prints the expected value.

## Working conventions

- Keep changes scoped; do not introduce a flake or root `default.nix` unless the task
  asks for it.
- Never hand-edit generated `version`/`urlMap`/`shaMap` values.
- Adding a new tool means adding `pkgs/<name>/default.nix` mirroring the existing one.
- Don't add markdown/docs files unless explicitly requested.

## Known gotchas

- Binaries are fetched, not compiled — a build only verifies the hash and packaging, not
  that upstream actually works.
- A function file like `pkgs/dir2mcp/default.nix` can't be built with a bare
  `nix-build pkgs/dir2mcp`; it needs `callPackage` (see above).
- Hashes are base32 (non-SRI) sha256 as emitted by GoReleaser; keep that format if you
  ever regenerate by hand.
